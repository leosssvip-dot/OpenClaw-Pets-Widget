# 和尚桌宠空闲动画改造进展

更新时间：2026-03-12

## 本轮目标

把和尚桌宠的 `idle` 状态改成更偏“禅定平静”的木鱼敲击动画，并让 `功德+1` 在敲击后以更克制的方式上浮。

## 设计结论

这次不走夸张 Q 弹路线，改成：

- 中慢节奏，单轮约 `1.78s`
- 重心稳定，动作主要集中在右臂、木槌、木鱼反馈
- 敲击前有轻微蓄势，敲击后有短暂余震和回静
- `功德+1` 不再像游戏掉字，而是偏暖金色、轻透明、短距离上浮

## 已完成实现

### 1. 和尚 idle 改为 GSAP 时间线

在 `DesktopPet.tsx` 中新增了和尚 `idle` 专用 GSAP timeline：

- `rest`
- `lift`
- `strike`
- `settle`

这样 idle 不再依赖原先那套偏机械的 CSS 敲击节奏，而是和 working 一样进入可控时间线。

### 2. 功德字幕和敲击点对齐

`MeritParticles` 增加了 `initialDelayMs`，现在可以把第一次 `功德+1` 的出现延迟到真正敲击之后，而不是组件激活就立刻冒字。

当前 idle 配置：

- 敲击周期：`1780ms`
- 首次功德浮现延迟：`1060ms`

这样视觉上更接近“敲一下，再浮一条功德”。

### 3. 功德字幕样式收敛

`styles.css` 中的 `功德+1` 样式已改成更克制的版本：

- 字号更小
- 颜色更柔和
- 发光更弱
- 上浮距离更短
- 左右漂移更轻

目标是让它像“香气般浮起”，而不是成就弹窗。

### 4. 修复木槌打不到木鱼、视觉像两根木棒

在你实际反馈之后，进一步定位到两个根因：

- 和尚右臂的 `sleeve` 和 `mallet` 在原 SVG 中本来就几乎平行重叠，容易看成两根棕色木棒
- 木槌旋转仍沿用了旧 rig 的锚点，导致摆动轨迹不对，视觉上无法真正敲到木鱼

本轮修复做了三件事：

- 重绘和尚右臂区域，让袖子只负责“手臂”，木槌单独成形
- 给木槌补上更明确的 `mallet-head`
- 把右臂和木槌的旋转原点改成当前 SVG rig 对应的实际坐标

这次修复的目标不是增加复杂度，而是把“结构错误”先纠正，不再用错误 rig 硬调参数。

### 5. 重新配平木鱼、手、槌子的构图关系

在结构修正后，继续出现“木鱼和手位置不自然、不和谐”的问题。进一步判断后，这不是单点坐标错误，而是整体构图失衡：

- 木鱼离身体太近，像贴在和尚身侧
- 持槌手位置偏低，和木鱼挤在一块
- 槌子路径太短，角色关系没有形成清晰的动作三角

这轮重新改成了更稳定的空间关系：

- 木鱼整体略向右外移
- 右手上提，回到更像“持槌准备落下”的位置
- 槌子改成更自然的斜向下落线
- 左手轻扶木鱼侧边，让主体关系更完整

目标是让画面从“几个元素挤在一起”变成“角色、工具、目标物件关系清楚”的一组动作构图。

### 6. 进一步把木鱼抬回自然工作区

在继续看实际效果后，又发现木鱼虽然不再和手挤在一起，但仍然偏低，像挂在右下角，不像和尚自然敲击木鱼时的工作区。

因此又做了一轮更明确的修正：

- 木鱼中幅上提，同时再略向右外移
- 左手从“扶住木鱼”改成“基本退出主关系”
- 右手持槌点继续上提，让敲击弧线更像从上往下轻落
- 木鱼、槽口、冲击环的命中坐标统一上移，避免动画仍围绕旧位置反馈

这一轮的目标是让画面关系更接近：

- 和尚主体在左
- 木鱼处在右侧腰上方工作区
- 右手负责主敲击动作
- 左手只作弱存在甚至接近退出

### 7. 降低右臂整体上扬角度

在继续看效果后，又发现右手和胳膊整体指向右上的角度仍然过大，接近明显举起的感觉，不像自然持槌待敲。

因此这轮做了更聚焦的动作姿态修正：

- 把右手静止持槌点整体下放
- 把待机姿态从偏高上扬收回到大约 `30°` 左右
- 同时压低 `idle` 与 `working` 的抬手峰值
- 保持木鱼工作区不变，只修正“手臂太抬”的问题

这轮不是再改构图，而是专门收敛右臂姿态，让角色看起来更放松，也更像持续稳定地敲木鱼。

## 技术说明

### 为什么改成 GSAP idle

因为这类角色动画要体现：

- 呼吸
- 预备
- 轻击
- 余震
- 回静

用统一 CSS keyframes 很难把这些阶段做得自然。GSAP timeline 更适合把动作拆成段落，并保持后续可调。

### 为什么不复杂化

这次没有引入新的动画引擎，也没有重做 SVG 结构，只是在现有和尚 rig 上重编排节奏。这样可以保持风险较低，也方便继续微调。

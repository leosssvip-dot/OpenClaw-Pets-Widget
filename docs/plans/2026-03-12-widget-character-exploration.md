# OpenClaw Widget Character Exploration

## Goal

- Define a unified mascot direction for OpenClaw desktop pets.
- Ensure every character can be implemented as simple 2D looped animation.
- Keep the style cute, cartoony, and product-aligned rather than illustrative or realistic.

## Locked Direction

- Style: flat chibi 2D cartoon
- Constraints:
  - no realism or semi-realism
  - no texture-heavy rendering
  - consistent head-to-body ratio
  - consistent eye / mouth / outline system
  - simple silhouettes suitable for small desktop sprite display
  - easy future rigging into separate animated parts

## Current Character Set

1. OpenClaw Lobster
2. Wooden Fish Monk
3. Key Penguin Courier
4. Scroll Turtle Butler

## Stitch Exploration Summary

- First pass exposed two problems:
  - style inconsistency
  - some characters became too realistic for desktop pet animation
- Second pass enforced:

  - flat shading only
  - unified palette family
  - simplified props and body shapes
- Latest generated direction is the current preferred lineup:
  - `OpenClaw Production Mascot Lineup`
  - focus on production-ready mascot consistency over novelty

## Animation Hooks

- Lobster: claw wave, antenna bounce, idle breathe
- Monk: wooden fish tap, robe sway, blink / chant bob
- Penguin: quick waddle, satchel bounce, salute wing

## Minimal Simplification Reset

  - background was not pure white
  - characters were still too complex
  - the style was still not minimal enough for tiny looping animation
- New hard constraints:

  - minimal Q-version cartoon

  - optimized for tiny desktop-pet loop animation

    - Key Penguin Courier
